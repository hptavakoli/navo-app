# context/product-source/snapshots/

**Frozen non-authority snapshots.** Each file in this folder is a dated, banner-prefixed snapshot of source material supplied to the project (for example, founder-supplied early-context documents). They are read-on-demand grounding only — not authority, not implementation instructions, not feature backlogs.

- Never edited in place. Revisions become new dated files alongside the original.
- Promotion into [`../../../docs/05-product-context.md`](../../../docs/05-product-context.md), [`../../../docs/06-feature-blueprint.md`](../../../docs/06-feature-blueprint.md), [`../../../docs/07-technical-direction.md`](../../../docs/07-technical-direction.md), ADRs at [`../../../docs/decisions/`](../../../docs/decisions/), or work-package task files happens only through an explicit planning conversation with the product owner — never by presence here.
- See the parent [`../README.md`](../README.md) for the broader non-authority rule and the [`CLAUDE.md`](../../../CLAUDE.md) hard rule it derives from.
