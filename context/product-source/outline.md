## How to use this outline

1. List the top-level sections that describe what this project is or could become — business idea, target users, feature concept, distinguishing approach, pricing thoughts, etc. Order them as the user thinks about the project, not as documentation hierarchy.
2. For each outlined section that warrants more than a paragraph, create `sections/NN-<topic>.md` with the corresponding content.
3. Update this outline as the idea evolves. The outline is editable; per-section files may grow long.

This document is not a feature spec. It is the user's working model of the project, captured in their own words.
