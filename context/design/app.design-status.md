# Navo App — Design status

**Status:** Scaffold. Activate when entering a design phase; until then, this file is a placeholder.

> Pointer-only status index for the design track. Records *where things stand right now* — current next-action, recently audited / refreshed briefs, deferred items, do-not-reopen decisions. Does **not** duplicate canonical content; the actual design owners live elsewhere in [`../design/`](../design/).

This file is the cold-start anchor for design-oriented sessions, the same way the milestone README is the anchor for development-oriented sessions.

---

## 1. Current next-action

> The single most important pointer in this file. What is the next concrete design action? Examples:
>
> - "Brand identity pre-flight review on `<file>` — pending product-owner sign-off."
> - "Component-library scope conversation — not started."
> - "No active design action; design track is paused."

Be specific. The cold-start session reads this and asks "what's next?" — the answer must be unambiguous.

## 2. Recently audited / refreshed briefs

## 3. Open Q-map

## 4. Deferred / future-horizon items

## 5. Do-not-reopen decisions
